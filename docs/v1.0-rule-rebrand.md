# v1.0 sub-project #1 — Rule rebrand + ID system

Status: **implemented** — every section below describes the surface that now lives on `main`. The CHANGELOG `[Unreleased]` block enumerates the concrete deliverables. Subsequent v1.0 sub-projects (source-layer rule expansion, PDF input + PDF layer, `--guide`, template-version detection) extend this foundation but do not re-litigate it.

## Goal

Migrate the v0.1 validation surface (9 errors + 2 warns, ad-hoc message strings) onto the v1.0 finding shape defined in [`spec-v1.0.md`](./spec-v1.0.md) §5: `UF-*` rule IDs, `must-fix` / `review` severity tiers, structured per-finding fields, and a versioned JSON schema. Foundational for every later v1.0 sub-project (rule expansion, PDF layer, `--guide` walkthrough).

## Scope

In scope:

1. New rule registry under `latex2ufdissertation/pipeline/rules.py`.
2. New `Finding` dataclass replacing free-form `errors: list[str]` + `warnings: list[str]` on `Issues`.
3. Rebrand every existing v0.1 check site to emit `Finding` records by `UF-*` ID instead of free-form strings.
4. JSON schema v1 frozen per [`spec-v1.0.md`](./spec-v1.0.md) §5 (`schema_version`, `findings: [Finding]`, `summary.exit_reason`, etc.).
5. Human-readable output rewritten to buffer-then-emit, grouped by layer + rule category, sorted for determinism.
6. Exit-code policy enforced via tier (`must-fix` count contributes; `review` does not), not via legacy boolean.
7. Replace the `tests/test_determinism.py` `xfail` placeholder with a real assertion: byte-identical `--json` output across two consecutive runs on `examples/demo_dissertation/`.

Out of scope (other v1.0 sub-projects):

- New rule checks beyond the 11 already in v0.1 (sub-project #2 — source-layer expansion).
- PDF input mode and PDF-layer checks (sub-project #3).
- `--guide` ETD walkthrough (later sub-project).
- Template-version detection beyond a stubbed null in `template_version` (later sub-project).

## Architecture

### Rule registry

`latex2ufdissertation/pipeline/rules.py` exports:

```python
@dataclass(frozen=True)
class Rule:
    id: str                # e.g. "UF-F13"
    severity: str          # "must-fix" | "review"
    layer: str             # "source" | "pdf" | "both"
    title: str             # short human label, e.g. "Document class"
    source_url: str        # canonical UF docs URL or in-repo anchor
    fix_hint: str | None   # short remediation; nullable until sub-project #2 fills it

RULES: dict[str, Rule] = { "UF-F1": Rule(...), ..., "UF-A2": Rule(...) }
```

One entry per `UF-*` ID listed in [`uf-rules.md`](./uf-rules.md). Single source of truth in code; metadata mirrors the catalog but is not generated from it (hand-synced — drift caught by a unit test that asserts every `UF-*` ID in `uf-rules.md` has a matching `Rule` entry).

### Finding type

`latex2ufdissertation/pipeline/types.py` gains:

```python
@dataclass(frozen=True)
class Finding:
    severity: str               # mirrors Rule.severity for fast filtering
    rule_id: str                # FK into RULES
    layer: str                  # source | pdf
    location: str               # "file.tex:42" (source) or "p.7" (pdf) or "" if global
    observed: str | None        # what the validator saw
    required: str | None        # what UF requires
    fix_hint: str | None        # from Rule, but per-finding override allowed
    source_url: str             # from Rule
```

The `Issues` collector replaces its `errors` / `warnings` lists with a single `findings: list[Finding]`. A helper `Issues.add(rule_id, *, location, observed=None, required=None, fix_hint=None)` resolves `severity`, `layer`, `source_url` from `RULES[rule_id]` so call sites stay terse.

### Output formatter

A new `latex2ufdissertation/pipeline/report.py` owns all user-facing emission. Two functions:

- `format_human(issues: Issues) -> str` — buffers all findings, sorts by `(layer, category_rank, rule_id, location)` (the category-rank tiebreaker is a human-readability concern that diverges intentionally from the spec's JSON sort key), groups by `layer` then by rule category prefix (`F` / `S` / `D` / `P` / `J` / `A`), renders each as `[severity] UF-XYZ location — observed / required (fix_hint)`.
- `format_json(issues: Issues) -> dict` — emits the spec §5 shape verbatim. Findings are sorted by the spec key `(layer, rule_id, location)`. `schema_version = "1.0"`. `template_version = None` for now (filled by future detection sub-project). `summary.exit_reason` is one of: `clean`, `must_fix_present`, `compile_failure`, `unsupported_template`, `unreadable_input`, `thesis_input`, `missing_toolchain`. `summary.exit_code` is derived from `summary.exit_reason` so the JSON payload never disagrees with the process exit state on fatal paths.

The CLI calls one or both depending on `--json`; nothing prints during check execution.

### Exit-code mapping

| Condition | Code | `exit_reason` |
|---|---|---|
| No `must-fix` findings | 0 | `clean` |
| ≥1 `must-fix` finding | 1 | `must_fix_present` |
| `ConverterError` (resolve/detect/compile) | 2 | `compile_failure` \| `unreadable_input` \| `unsupported_template` \| `thesis_input` |
| `lualatex` missing | 3 | `missing_toolchain` |

Exit-code policy lives in `cli.py`; the precise `exit_reason` per exception type is set wherever the exception is raised.

### v0.1 → UF-* mapping (finalized)

Every v0.1 emit site in `checks.py` is accounted for:

| v0.1 site | Condition | New rule | Severity |
|---|---|---|---|
| L45 (E1) | documentclass not `ufdissertation` | UF-F13 | must-fix |
| L50 (E2–E5) | one of `\title` / `\author` / `\degreeType` / `\thesisType` missing | UF-F14 | must-fix |
| L56 (E6–E9) | `\setAcknowledgementsFile` / `\setAbstractFile` / `\setReferenceFile` / `\setBiographicalFile` not set | UF-F8 | must-fix |
| L60 (E6–E9) | `\set*File` is set but the target file does not exist on disk | UF-P1 | must-fix |
| L64 (W1) | `editMode` option present in `\documentclass` | UF-D1 | review |
| L68 (W2) | `% !TEX program = pdflatex` or `xelatex` hint | UF-D2 | must-fix (per spec rebrand) |

Note the E6–E9 family splits into **two** rule IDs depending on which condition fires: `UF-F8` when the `\set*File` macro is absent, `UF-P1` when the macro is set but its target is missing. The split mirrors the rule catalog: F8 is about presence of the page-order macros; P1 is about filesystem companions to the macros.

The severity bump for the LuaLaTeX hint (W2 → must-fix) is per [`uf-rules.md`](./uf-rules.md) UF-D2 (`Severity: must-fix`). v0.1 emitted it as a warn; the rebrand promotes it to must-fix.

## Data flow

```
input ──► resolve ──► detect master .tex ──► run_checks (Source layer only in v0.1)
                                                  │
                                                  ▼
                                      issues.add(rule_id, location, ...)
                                                  │
                                                  ▼
                                  Issues.findings: list[Finding]
                                                  │
                       ┌──────────────────────────┼──────────────────────────┐
                       ▼                                                     ▼
              format_human(issues)                                  format_json(issues)
                       │                                                     │
                       ▼                                                     ▼
                    stderr                                            stdout (--json)
```

Severity-based exit code is computed from `issues.findings` after all checks run; no early-exit on first must-fix.

## Determinism

Three sources of non-determinism in v0.1 to neutralize:

1. **Emission order** — checks print as they run; order depends on dict iteration in some paths. Fix: buffer findings, sort `(layer, rule_id, location)` before emission.
2. **Temporary paths** — already normalized in v0.1.
3. **Schema field order** — `json.dumps(..., sort_keys=True)` for both the top-level object and each finding.

Pinning test in `tests/test_determinism.py` runs `main(["--json", "examples/demo_dissertation/", "--dry-run"])` twice, captures `--json` stdout, asserts byte equality.

## Testing strategy

Three-commit cadence (`test:` red → `feat:`/`refactor:` green → `chore:` rollup):

1. **Registry drift test** — assert every `UF-*` ID mentioned in `docs/uf-rules.md` has a matching `Rule` entry, and vice versa.
2. **Finding shape test** — assert `Finding(...)` has all 8 spec-§5 fields, with the right nullability.
3. **Per-rebrand tests** — for each of the 11 existing checks, a focused test asserts that running on a broken-input fixture produces a `Finding` with the correct `rule_id` + `severity` + `layer` + non-empty `location`.
4. **JSON schema test** — assert `format_json(issues)` produces a dict whose top-level keys are exactly `{schema_version, input, template_version, findings, summary}`; assert `summary` keys are exactly `{must_fix_count, review_count, exit_code, exit_reason}`.
5. **Human-output snapshot** — golden-file test on the demo fixture's report (expected: empty findings, clean summary).
6. **Determinism pinning** — see above.
7. **Coverage** — rebrand should raise cli.py + types.py coverage; floor ratchets up at the end of this sub-project (target after this sub-project: ≥70%).

## Rules to follow

- TDD three-commit cadence per change.
- Conventional Commits: likely sequence `feat:` (registry + Finding type) → `refactor:` (rebrand each emit site) → `feat:` (report formatter + schema) → `test:` (determinism pinning).
- Stdlib only at runtime; no new deps in `pyproject.toml`.
- Single dependency direction: `rules.py` → `types.py` → `checks.py` → `report.py` → `cli.py`. No backward edges.
- One responsibility per file.
- Hard break of v0.1 output. v1.0 is a major bump; no compat shim.
- No `print()` inside checks. Emission is the report formatter's job only.
- Public API contract: `__all__` in `latex2ufdissertation/__init__.py` enumerates `Issues`, `Finding`, `Rule`, `RULES`, `run_checks`, `__version__`. Nothing else is supported.
- No magic strings — severity, layer, exit_reason literals are constants in `rules.py`.
- No AI/agent mentions in any committed artifact.

## Constraints

- **Python 3.10+ stdlib only.** Dataclasses, json, typing only.
- **Pre-commit + CI green** before push.
- **Coverage floor.** Currently 60%; ratchet up to ≥70% at end of this sub-project.
- **Schema frozen at sub-project completion.** After this lands, future sub-projects extend the schema only via additive nullable fields. No renames, no removals, no required-field additions without a major-version bump.
- **Determinism enforced.** Two runs → byte-identical `--json`. Real pinning test, no xfail.
- **Offline.** No network in any code path this sub-project touches.
- **Spec is the source of truth.** If a section of [`spec-v1.0.md`](./spec-v1.0.md) §5/6/7 conflicts with this design, the spec wins.

## Sequencing
